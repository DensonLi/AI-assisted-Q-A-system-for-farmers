# 《中国农户种植用户研究报告》

> **专家组**：农业种植专家 + 农业行业数字化专家
> **版本**：v1.0　**日期**：2026-04

---

## 一、研究目标

摸清中国不同农业区主要作物的**种植时序、典型问题、信息化痛点**，为产品功能设计提供依据。回答三个核心问题：

1. **谁在什么时间、什么地区种植什么作物？**
2. **种植过程中会遇到哪些典型问题？**
3. **他们需要什么样的信息化支持？**

---

## 二、中国农业分区与主要作物时序

按照农业农村部划分，将全国分为 **7 大农业生态区**。对每个区标注主要作物与关键农时窗口（月份为粗略范围，具体按物候期调整）。

## 三、农户典型问题分类（高频 Top 7）

基于农技推广站、12316 三农热线咨询数据分析，农户问题可归为 7 大类：

| 问题类别 | 占比 | 典型提问举例 |
|---------|------|------------|
| **1. 病虫害识别与防治** | 约 35% | "我这黄瓜叶子上有黄斑是啥病？""稻飞虱用什么药？" |
| **2. 施肥与营养管理** | 约 18% | "玉米抽雄期追什么肥？""叶子发黄是缺啥？" |
| **3. 农时与物候决策** | 约 12% | "今年春旱啥时候播种合适？""小麦啥时候浇返青水？" |
| **4. 品种选择** | 约 10% | "我们这地块选哪个玉米品种产量高？" |
| **5. 气象灾害应对** | 约 8% | "明天要下暴雨咋办？""寒潮怎么防冻？" |
| **6. 农药/化肥使用规范** | 约 8% | "这个药能和那个一起喷吗？""安全间隔期多久？" |
| **7. 销售与政策** | 约 9% | "补贴怎么申请？""今年粮价咋样？" |

本产品**优先覆盖 1-6 类**（技术类问题），第 7 类作为后续扩展。

## 四、用户画像（Persona）

### P1：种粮大户（核心用户）

- **年龄**：35-55 岁，男性为主
- **规模**：承包 100-2000 亩
- **数字化程度**：智能手机熟练，微信重度使用
- **痛点**：
  - 面积大，病虫害排查耗时
  - 需要快速决策，不能等农技员上门
  - 经验丰富但面对新品种/新药剂信息不全
- **需求**：**图文结合的快速诊断 + 可靠的用药方案 + 持续跟踪**

### P2：小农户（基础用户）

- **年龄**：45-65 岁
- **规模**：5-30 亩自家地
- **数字化程度**：会用微信，文字输入慢，偏好语音
- **痛点**：
  - 种植经验主要来自口耳相传
  - 不会使用复杂 App
- **需求**：**语音交互 + 方言兼容 + 简单的步骤指引**

### P3：新型职业农民 / 返乡创业者（高价值用户）

- **年龄**：25-40 岁
- **规模**：合作社、家庭农场
- **数字化程度**：高，熟悉数字化工具
- **痛点**：
  - 缺乏实操经验，理论要转为实践
  - 需要跨区域（从城市回县乡）对接本地农技资源
- **需求**：**系统化知识 + 本地专家对接 + 数据记录分析**

### P4：农技员 / 合作社技术员（B 端使用者）

- **年龄**：30-50 岁
- **角色**：辅助多户农民
- **需求**：**批量查询 + 跨地块对比 + 知识沉淀**

## 五、信息化现状与痛点

### 5.1 现有工具

| 工具类型 | 代表 | 局限 |
|---------|------|------|
| 农技 App | 云上智农、农管家 | 内容泛、无个性化、问答能力弱 |
| 微信公众号/短视频 | 各地农技站 | 单向推送，不解决具体问题 |
| 12316 热线 | 政府服务 | 服务时间有限，专家数量不足 |
| 传统 AI 客服 | 部分电商 | 无农业专业知识，答非所问 |

### 5.2 核心未被满足的需求

1. **"我的地块、我的作物"的个性化问答**
   - 现有系统要么是广播式知识，要么要求反复输入上下文
2. **跨会话的持续记忆**
   - 上次问"玉米苗黄苗怎么办"，下次问"追肥量"时需要重新解释地块情况
3. **用户经验的沉淀**
   - 用户 3 年种下来的经验（"我这块地偏酸""去年伏旱严重"）应该被系统记住
4. **可校验的权威建议**
   - 农民担心"AI 瞎说"，需要引用来源与安全兜底（如农药剂量）

## 六、研究结论与对产品的启示

| 研究发现 | 产品启示 |
|---------|---------|
| 问题高度依赖**地区 + 作物 + 农时** | 对话需绑定 **区域/作物上下文** |
| 农民积累的地块经验宝贵但无处存放 | 需要**可持续的用户记忆系统** |
| 经验更新频繁（品种换了、改了施肥习惯） | 记忆需支持**增量更新 + 用户确认** |
| 老年用户文字输入慢 | 需要**语音输入 + 快捷按钮** |
| 用药、施肥错误代价大 | 关键建议需**引用知识库来源**、**安全兜底** |
| 图片是最有效的问题描述 | 远期支持**图片识别**（本期不实现） |

这些洞察直接驱动了下一步的功能设计（详见 `02_产品功能设计.md`）。
