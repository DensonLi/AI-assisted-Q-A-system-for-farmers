```

 │                     Nginx (反向代理)                         │

 └──────────────────────┬──────────────────────────────────────┘

 ┌──────────────────────▼──────────────────────────────────────┐
 │                   FastAPI Backend                           │
 │  ┌──────────────┬──────────────┬─────────────────────────┐ │
 │  │  Auth 模块   │  对话/问答    │  记忆管理 / 农时提醒    │ │
 │  └──────┬───────┴──────┬───────┴────────┬────────────────┘ │
 │         │              │                 │                  │
 │  ┌──────▼──────────────▼─────────────────▼──────────────┐  │

      │Postgres│      │  Redis    │   │ 外部 API      │
      │(主数据)│      │(缓存+限流)│   │ ─ 知识库      │

### 技术栈决策

| 层 | 选型 | 理由 |
|----|------|------|
| 前端 | React 18 + Vite + Ant Design | 组件完善、中文生态好、农户适用性 |
| 状态 | Zustand + TanStack Query | 简单、天然支持服务端缓存 |
| 后端 | FastAPI (Python 3.12) | 异步高性能、类型安全、OpenAPI 文档 |
| ORM | SQLAlchemy 2.0 (async) | 与 FastAPI 异步完美配合 |
| 数据库 | PostgreSQL 16 | JSONB 支持记忆条目、全文搜索 |
| 缓存 | Redis 7 | 会话缓存、限流、LLM 响应缓存 |
| 认证 | JWT (access + refresh) | 无状态，便于横向扩展 |
| 部署 | Docker + docker-compose | 满足"易于打包"要求 |

## 二、数据模型设计

### 2.1 实体关系图（ERD）

```
users ─┬──< conversations ─< messages
       │
       ├──< user_crop_memories ─< memory_items
       │
       └── profile (region_id: 常住区域)

## 三、核心业务流程

### 3.1 问答流程（ConversationOrchestrator）

```
用户提问 (question, conversation_id)
    │
    ▼
┌─ 1. 加载对话上下文 ─────────────────────────┐
│  ─ conversation.region_id / crop_id        │
│  ─ 最近 N 轮消息                            │
└────────────────────────────────────────────┘
    │
    ▼
┌─ 2. 加载长期记忆 ──────────────────────────┐
│  SELECT * FROM memory_items                 │
│  WHERE memory.(user, region, crop) 匹配     │
│    AND status = 'active'                    │
└────────────────────────────────────────────┘
    │
    ▼
┌─ 3. 计算当前物候期 ────────────────────────┐
│  PhenologyService.current_stage(region, crop, today) │
└────────────────────────────────────────────┘
    │
    ▼
┌─ 4. 检索知识库 (RAG) ──────────────────────┐
│  KnowledgeClient.search(question, crop, region) │
│  → 取 Top-K 相关文档                        │
└────────────────────────────────────────────┘
    │
    ▼
┌─ 5. 组装 Prompt 并调用 LLM ────────────────┐
│  system:  专业种植顾问角色 + 安全约束       │
│  context: 区域/作物/物候期/长期记忆/检索片段 │
│  history: 最近对话                          │
│  user:    本次提问                          │
│  →  LLM 生成回答 + 可能的记忆候选            │
└────────────────────────────────────────────┘
    │
    ▼
┌─ 6. 提取记忆候选 ───────────────────────────┐
│  MemoryService.extract_from_turn(...)       │
│  若检测到新事实，生成 memory_update_proposal │
└────────────────────────────────────────────┘
    │
    ▼
┌─ 7. 生成追问建议 ───────────────────────────┐
│  LLM 提示 "请给出 3 个相关追问"              │
└────────────────────────────────────────────┘
    │
    ▼
 返回:
 {
   answer, references, suggestions,
   proposed_memory_updates: [...]
 }
```

### 3.2 记忆提取策略

使用 **LLM Function Calling** 让模型在回答同时产出结构化记忆候选。

```python
tools = [
  {
    "name": "propose_memory_update",
    "description": "当从用户回答中提取到地块新事实时调用",
    "input_schema": {
      "properties": {
        "key":   { "enum": ["soil_type","variety","irrigation",
                            "pest_history","fertilizer_habit","terrain"] },
        "value": { "type": "string" },
        "action": { "enum": ["add", "update"] },
        "reason": { "type": "string" }
      }
    }
  }
]
```

**安全约束**：
- 每次最多提出 3 条候选（防止 LLM 过度推断）
- `confidence < 0.6` 的直接丢弃
- 候选写入 `memory_update_proposals.status = pending`
- 前端展示弹窗 → 用户确认 → 后端调 `/memories/proposals/{id}/accept` 写入 active

### 3.3 物候期计算

简化为"按农业区 × 作物 × 日期"查询：

```python
def current_stage(region: Region, crop: Crop, today: date) -> PhenologyStage | None:
    return db.query(PhenologyStage).filter(
        agro_zone == region.agro_zone,
        crop_id == crop.id,
        (start_month, start_day) <= (today.month, today.day) <= (end_month, end_day)
    ).first()
```

数据由 `scripts/seed-data.py` 批量录入。

## 四、API 设计（新增/变更端点）

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/v1/regions/tree` | 区域树（分级加载） |
| GET | `/api/v1/regions/search?q=` | 区域搜索 |
| GET | `/api/v1/crops/tree` | 作物分类树 |
| GET | `/api/v1/crops/popular` | 热门作物 |
| POST | `/api/v1/conversations` | 新建对话（body: region_id, crop_id, title） |
| POST | `/api/v1/conversations/{id}/ask` | 提问（原 /ask 迁移至此） |
| GET | `/api/v1/conversations/{id}/suggestions` | 追问建议 |
| GET | `/api/v1/memories?region_id=&crop_id=` | 获取某组合的记忆 |
| PUT | `/api/v1/memories/items/{id}` | 编辑单条记忆 |
| DELETE | `/api/v1/memories/items/{id}` | 删除单条记忆 |
| GET | `/api/v1/memories/proposals?status=pending` | 待确认的记忆提案 |
| POST | `/api/v1/memories/proposals/{id}/accept` | 接受提案 |
| POST | `/api/v1/memories/proposals/{id}/reject` | 拒绝提案 |
| GET | `/api/v1/phenology/current?region_id=&crop_id=` | 当前物候期 |

JWT 鉴权保持不变。

## 五、缓存与性能策略

| 场景 | 缓存层 | TTL |
|------|--------|-----|
| 区域树 | Redis | 24h |
| 作物分类 | Redis | 24h |
| 物候期（按区×作×日期） | Redis | 6h |
| 知识库检索结果 | Redis（按 hash(question) 键） | 10min |
| LLM 同问题命中 | Redis（按 hash(prompt) 键） | 5min |

**限流**：
- 每用户每分钟最多 20 次问答调用
- 未登录接口不对外

## 六、扩展性考虑

- **水平扩展**：FastAPI 无状态，可起多副本 + Nginx 负载均衡
- **LLM 切换**：`LLMClient` 抽象接口，支持 Claude / 通义千问 / 百川 / 本地 Ollama
- **知识库切换**：`KnowledgeClient` 接口化，支持 RESTful / Dify / LangChain
- **国际化**：i18n 预留（本期仅中文）

## 七、安全架构

| 方面 | 措施 |
|------|------|
| 密码存储 | bcrypt（cost=12） |
| Token | JWT，access 60min + refresh 7d，refresh 轮转 |
| 传输 | HTTPS 强制（生产） |
| SQL 注入 | SQLAlchemy ORM 参数化查询 |
| XSS | React 自动转义 + CSP 头 |
| CORS | 白名单配置 |
| 速率限制 | Redis + slowapi |
| 敏感词 | LLM 输出侧过滤（农药超量警告） |
| 日志 | 脱敏处理，不记录密码/token |
