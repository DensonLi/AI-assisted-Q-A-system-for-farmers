# 默认账号与密码

> ⚠️ **安全提示**：本文档含敏感信息，请妥善保管，不要上传至公开代码仓库。  
> 生产环境部署前，**必须修改所有默认密码**。

---

## 一、应用系统账号

### 1. 管理员账号（系统内置）

| 项目 | 默认值 |
|------|--------|
| 用户名 | `admin` |
| 密码 | `Admin@123456` |
| 邮箱 | `admin@example.com` |
| 权限 | 管理员（可管理用户、查看系统配置） |

**登录地址**：`http://服务器IP/`（首页即登录页）

**作用说明**：
- 系统首次启动时自动创建
- 可进入「用户管理」页面新增/禁用普通用户
- 可进入「系统配置」页面在线修改 LLM API Key、知识库配置，无需重启服务

**修改方式**：登录后点击右上角用户名 → 「修改密码」；或在 `.env` 文件中修改 `FIRST_ADMIN_PASSWORD` 后重新部署。

---

### 2. 普通用户账号

普通用户由管理员在「用户管理」页面手动创建，系统**无内置普通用户**。

| 权限 | 说明 |
|------|------|
| 问答对话 | 可新建对话、提问、查看历史 |
| 长期记忆 | 查看和管理自己的地块记忆 |
| 农事日历 | 查看和管理自己的提醒事件 |
| 用户管理 | ❌ 不可访问（仅管理员） |
| 系统配置 | ❌ 不可访问（仅管理员） |

## 二、数据库账号

### PostgreSQL 数据库

| 项目 | 默认值 | 说明 |
|------|--------|------|
| 用户名 | `postgres` | 超级管理员账号 |
| 密码 | `.env` 中自行设置 | 无内置默认密码，部署时必须配置 |
| 数据库名 | `farming_qa` | 应用专用数据库 |
| 端口 | `5432`（仅容器内） | 生产环境不对外暴露 |
| 主机（容器内） | `postgres` | Docker 内部服务名 |

**连接字符串格式**：
```
postgresql://postgres:你的密码@postgres:5432/farming_qa
```

**登录数据库命令**（在服务器上执行）：
```bash
docker compose exec postgres psql -U postgres farming_qa
```

**作用说明**：
- 存储所有业务数据：用户、对话、消息、地块记忆、农事提醒、系统配置
- 数据持久化到 Docker 数据卷 `postgres_data`，容器重启数据不丢失

## 四、系统安全配置

### JWT 签名密钥

| 项目 | 说明 |
|------|------|
| 配置位置 | `.env` → `SECRET_KEY` |
| 作用 | 用于签发和验证用户登录 Token |
| Token 有效期 | 60 分钟（`ACCESS_TOKEN_EXPIRE_MINUTES`） |
| 默认值 | **无默认值，部署前必须设置** |

**生成安全密钥**：
```bash
openssl rand -hex 32
```

> ⚠️ 修改 `SECRET_KEY` 后，所有已登录用户的 Token 立即失效，需重新登录。

## 五、一键查看当前配置

在服务器项目目录下执行，可查看当前生效的所有配置（API Key 会被脱敏显示）：

```bash
# 查看非敏感配置
grep -v "KEY\|PASSWORD\|SECRET" .env

# 查看是否配置了 LLM（显示前4位）
grep "LLM_API_KEY" .env | cut -c1-20
```

## 六、生产部署前必改清单

| 配置项 | 位置 | 要求 |
|--------|------|------|
| `POSTGRES_PASSWORD` | `.env` | 改为强密码（16位以上，纯字母数字） |
| `SECRET_KEY` | `.env` | 用 `openssl rand -hex 32` 生成 |
| `FIRST_ADMIN_PASSWORD` | `.env` | 改为强密码，首次登录后再次修改 |
| `LLM_API_KEY` | `.env` | 填入真实 API Key |
| `FIRST_ADMIN_EMAIL` | `.env` | 改为真实邮箱 |
