## 2. 配置清单

将 `.env.example` 复制为 `.env` 并填写以下配置：

```bash
cp .env.example .env
```

### 必填项

| 变量 | 说明 | 示例 |
|------|------|------|
| `DATABASE_URL` | PostgreSQL 同步 URL | `postgresql://user:pass@db:5432/farming` |
| `ASYNC_DATABASE_URL` | PostgreSQL 异步 URL | `postgresql+asyncpg://user:pass@db:5432/farming` |
| `SECRET_KEY` | JWT 签名密钥（≥32位随机字符串）| `openssl rand -hex 32` 生成 |
| `FIRST_ADMIN_USERNAME` | 初始管理员用户名 | `admin` |
| `FIRST_ADMIN_EMAIL` | 初始管理员邮箱 | `admin@yourorg.com` |
| `FIRST_ADMIN_PASSWORD` | 初始管理员密码（≥8位）| `Admin@2026` |
| `LLM_API_KEY` | Anthropic API Key | `sk-ant-...` |

### 可选项

| 变量 | 默认值 | 说明 |
|------|--------|------|
| `LLM_MODEL` | `claude-sonnet-4-5` | LLM 模型版本 |
| `LLM_MAX_TOKENS` | `1500` | 单次最大输出 token |
| `LLM_TEMPERATURE` | `0.3` | 温度（0=确定性，1=创意性）|
| `LLM_HISTORY_TURNS` | `5` | 携带的历史对话轮数 |
| `KNOWLEDGE_API_BASE_URL` | `""` | ziiku.cn 知识库完整接口地址 |
| `KNOWLEDGE_API_KEY` | `""` | ziiku.cn x-api-key 认证密钥 |
| `KNOWLEDGE_BOT_ID` | `""` | ziiku.cn 机器人 ID（bot_id）|
| `REDIS_URL` | `redis://redis:6379/0` | Redis 连接串 |
| `DEBUG` | `false` | 调试模式（生产设为 false）|

## 3. 本地开发环境

### 3.1 初次启动

```bash
# 克隆代码
git clone <your-repo-url>
cd 农户种植技巧AI辅助问答系统

# 创建本地配置
cp .env.example .env.local
# 编辑 .env.local，填写必要配置

# 一键初始化开发环境（Python venv + npm install）
chmod +x scripts/setup-dev.sh
./scripts/setup-dev.sh
```

### 3.2 日常开发

```bash
# 启动基础设施（PostgreSQL + Redis）+ 后端 + 前端
./scripts/start-dev.sh

# 停止所有服务
./scripts/stop-dev.sh
```

访问地址：
- 前端：http://localhost:5173
- 后端 API：http://localhost:8000
- API 文档：http://localhost:8000/docs

## 4. 数据库初始化

### 4.1 运行 Alembic 迁移

```bash
cd backend
# 激活虚拟环境
source .venv/bin/activate

# 执行全部迁移
alembic upgrade head
```

### 4.2 写入种子数据


## 5. Docker 生产部署

### 5.1 构建并启动

```bash
# 复制生产配置
cp .env.example .env
# 编辑 .env，填写所有必填项（特别是 SECRET_KEY 和 LLM_API_KEY）

# 构建镜像并启动（首次约需 3-5 分钟）
docker compose up -d --build

# 查看服务状态
docker compose ps

# 查看日志
docker compose logs -f backend
docker compose logs -f frontend
```

### 5.2 运行数据库迁移（首次部署或升级）

```bash
docker compose exec backend alembic upgrade head
docker compose exec backend python ../scripts/seed_data.py
```

### 5.3 服务地址

| 服务 | 内部地址 | 外部暴露 |
|------|---------|---------|
| 前端（Nginx） | frontend:80 | 0.0.0.0:80 |
| 后端（FastAPI）| backend:8000 | 仅内部，由 Nginx 代理 |
| PostgreSQL | db:5432 | 不暴露 |
| Redis | redis:6379 | 不暴露 |

### 5.4 反向代理（可选）

如果在 Nginx 容器前还有一层外部 Nginx 或 Caddy，参考以下配置：

```nginx
server {
    listen 443 ssl;
    server_name your-domain.com;

    ssl_certificate /path/to/cert.pem;
    ssl_certificate_key /path/to/key.pem;

    location / {
        proxy_pass http://127.0.0.1:80;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-Proto https;
    }
}
```

## 7. 日常维护操作

### 7.1 升级版本

```bash
git pull origin main
docker compose up -d --build
docker compose exec backend alembic upgrade head
```

### 7.2 备份数据库

```bash
# 导出备份
docker compose exec db pg_dump -U postgres farming > backup_$(date +%Y%m%d).sql

# 恢复备份
cat backup_20260426.sql | docker compose exec -T db psql -U postgres farming
```

### 7.3 查看系统日志

```bash
# 后端实时日志
docker compose logs -f backend

# 查看最近 200 行
docker compose logs --tail=200 backend
```

### 7.4 重置特定用户密码

```bash
# 方法1：通过管理界面（推荐）
# 以 Admin 身份登录 → 用户管理 → 重置密码

# 方法2：通过 API
curl -X POST http://localhost:8000/api/v1/users/{user_id}/reset-password \
  -H "Authorization: Bearer <admin_token>" \
  -H "Content-Type: application/json" \
  -d '{"new_password": "TempPass@2026"}'
```

## 8. 常见问题排查

### Q1：后端启动报 `Could not connect to database`

```bash
# 检查 PostgreSQL 是否运行
docker compose ps db

# 检查连接串是否正确
echo $ASYNC_DATABASE_URL

# 手动测试连接
docker compose exec db psql -U postgres -d farming -c "SELECT 1"
```

### Q2：LLM 回答为纯文本（无记忆提案）

原因：`LLM_API_KEY` 未配置或配额不足。

```bash
# 检查配置
docker compose exec backend env | grep LLM

# 测试 API 连通性
curl https://api.anthropic.com/v1/messages \
  -H "x-api-key: $LLM_API_KEY" \
  -H "anthropic-version: 2023-06-01" \
  -H "content-type: application/json" \
  -d '{"model":"claude-haiku-4-5","max_tokens":10,"messages":[{"role":"user","content":"hi"}]}'
```

### Q3：区域/作物选择器为空

原因：种子数据未写入。

```bash
docker compose exec backend python ../scripts/seed_data.py
```

### Q4：前端登录后跳转空白页

可能是前端构建缓存问题：

```bash
docker compose stop frontend
docker compose rm frontend
docker compose up -d frontend
```

### Q5：bcrypt 相关 ValueError

原因：`bcrypt>=5.0.0` 与 `passlib 1.7.4` 不兼容。确保 `requirements.txt` 中固定 `bcrypt==4.0.1`。

```bash
pip install "bcrypt==4.0.1"
```
