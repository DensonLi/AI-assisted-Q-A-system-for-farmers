## 一、产品定位

**一句话定位**：绑定"地块-作物-农户经验"的 AI 种植顾问，让每个农户都有一位懂他家地块的数字农技员。

### 价值主张

- **个性化**：所有问答基于**用户的区域 + 作物 + 历史经验**，不是泛泛的百科回答
- **连续性**：跨会话记住用户地块特征、品种、习惯
- **可控性**：AI 更新对用户的"印象"前必须征得确认，用户随时可查看/编辑/删除记忆
- **权威性**：关键建议引用知识库来源，涉及安全的建议强制兜底

## 二、核心信息架构

### 2.1 三层上下文模型

```
┌─────────────────────────────────────────────┐
│  L1 长期记忆 (UserProfile + CropMemory)     │  跨会话持久化
│  ─ 常住区域                                  │
│  ─ 每个 (区域, 作物) 组合的地块特征/品种/习惯 │
├─────────────────────────────────────────────┤
│  L2 会话上下文 (Conversation)                │  当前对话内的历史消息
│  ─ 绑定：区域 + 作物                         │
│  ─ 20 轮滚动窗口                             │
├─────────────────────────────────────────────┤
│  L3 即时问题 (Current Question)              │  本轮提问
└─────────────────────────────────────────────┘
```

LLM 生成回答时，按优先级注入：系统指令 → L1 长期记忆 → L2 会话上下文 → L3 本次提问。

### 2.2 记忆粒度

**用户记忆以 `(用户, 区域, 作物)` 为最小粒度**。例如用户张三在：
- `(山东潍坊, 小麦)`：土壤偏碱、用"济麦44"、偏好深施基肥
- `(山东潍坊, 大蒜)`：种"金乡蒜"、每年 10 月播

两份记忆互不干扰。切换作物即切换记忆体。

### 2.3 记忆条目结构

每条记忆是一个事实性陈述 + 元数据：

```
{
  "key": "土壤类型",
  "value": "偏碱性，pH 7.8",
  "confidence": 0.9,         // 0-1
  "source": "user_confirmed" | "ai_inferred",
  "created_at": "2026-03-15",
  "last_verified_at": "2026-04-20"
}
```

常见记忆键：土壤类型、品种、播种方式、灌溉条件、历年病虫害、用药习惯、地块地形等。

## 四、关键用户流程

### 4.1 首次使用流程

```
注册/登录
  │
  ▼
(首次) 引导设置常住区域 ──► 跳过
  │
  ▼
主页：进入对话
  │
  ▼
[选择区域] [选择作物]  (已绑定则默认带入)
  │
  ▼
进入对话：显示农时提醒 + 问题快捷入口
  │
  ▼
用户提问 → AI 回答 → 智能追问建议
  │
  ▼
(AI 识别到新事实) 弹窗："要记住'您的地块土壤偏碱'吗？" 
  │
  ▼
用户确认 → 写入长期记忆
```

### 4.2 持续使用流程（核心）

**场景**：李师傅上次问过玉米苗期施肥，两周后再问"现在浇水合适吗？"

```
1. 李师傅登录 → 选择 (河北沧州, 夏玉米) → 新建对话
2. 系统自动注入记忆：
   ── "沧州夏玉米"（上下文）
   ── "沙壤土、滴灌、郑单958"（李师傅的长期记忆）
   ── "本周大喇叭口期"（物候期）
3. 李师傅提问："现在浇水合适吗？"
4. AI 基于完整上下文回答，无需反复确认基础信息
5. AI 回答中推断出"这段时间当地无降雨"，弹窗问：
   "看起来您最近几天没有降雨，需要记录到地块信息吗？"
6. 李师傅确认 → 记忆更新
```

### 4.3 记忆更新流程（关键交互）

触发时机：
- AI 在用户回答中识别到**新事实**或**与现有记忆冲突**
- 用户主动编辑记忆页

更新前必须**明示用户**并**等待确认**：

```
┌───────────────────────────────────────┐
│  🌾 要记住这些地块信息吗？              │
│                                        │
│  根据您刚才的描述，我想记录以下信息：    │
│                                        │
│  ✓ 土壤类型：偏碱性（新增）             │
│  ⚠ 品种：郑单958 → 先玉335（更新）     │
│    (原记录 2025-09-12)                 │
│                                        │
│  [全部保存]  [逐条选择]  [不保存]       │
└───────────────────────────────────────┘
```

## 六、交互规范要点

| 规范 | 说明 |
|------|------|
| **所有修改用户记忆的操作需显式确认** | 包括 AI 自动推断触发的更新 |
| **老年用户友好** | 字号 ≥ 14px、对比度 ≥ AA、关键按钮 ≥ 48×48px 触达区 |
| **快捷操作优先于输入** | 病虫害、施肥、灌溉、灾害应对 4 个一级入口 |
| **语气克制** | 不使用"最好的""完美的"等主观词；提供方案 + 原因 + 风险 |
| **安全兜底** | 用药建议必须附带"安全间隔期/剂量上限" |
| **移动端优先** | 触屏操作、单手可达 |

## 七、对功能开发的影响

基于本设计，系统必须在数据模型、API 和前端上新增：

**数据模型**：`regions`、`crops`、`user_crop_memories`、`memory_items`、`conversation` 增加 `region_id/crop_id`
**API**：区域/作物列表、记忆 CRUD、记忆确认端点、农时提醒端点
**前端**：区域/作物选择器组件、记忆确认弹窗、记忆管理页、农时提醒卡片
